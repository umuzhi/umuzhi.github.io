判断自己电脑是否和远程的key是否连接成功

打开终端，输入：

ssh -T git@github.com

bash 会显示：

The authenticity of host 'github.com (207.97.227.239)' can't be established.

RSA key fingerprint is ......

Are you sure you want to continue connecting (yes/no)?

输入yes 会出现如下提示：

Hi username! You've successfully authenticated, but GitHub does not provide shell access.

如果成功看见此提示，说明你已经配置好了，nice！

Git远程仓库连接问题

Hexo远程部署ERROR:

Repository not found. fatal: Could not read from remote repository.

其实我在这里卡的时间最久了，检查了ssh key，和相关问题最后才发现是我没有在GitHub上建立自己的项目，所以导致不能传输到对应的仓库。

创建一个仓库，名字以自己的github名字开头,即yourName.github.io，比如我的就是umzhi.github.io

因为这个我折腾了几个小时，所以Git的使用文档一定要仔细看，再次说明英文也很重要。

写博客

使用Git Bash定位到我们的hexo根目录，执行命令：

$ hexo n "距离2018年还有45天" ，效果如下图所示：

当你看到该命令时：

INFO Created: E:\4_workplace\3_git\1_web\1_hexo\muzhi\source\_posts\距离2018年还有45天.md

证明你已经生成了新的这篇文章，注意我这里的hexo n "博客名称" 是缩写，正常来说应该是hexo new "博客名称"。

比如说我的这篇文章就存在于E:\4_workplace\3_git\1_web\1_hexo\muzhi\source\_posts 该目录下，你生成的博客一般都储存在hexo/source/_posts下，我的文件夹就是muzhi\source\_posts下。

Hexo标签问题

由于开始并不熟悉Hexo的标签管理，使用的是最初级的方法，tag: tag1 ，后来查阅官方文档和其他资料，发现正确的写法应该是如下：注意冒号后面有空格，Hexo对空格很敏感的哟！

tags:

- PS3

- Games

或者是

tag: [tag1,tag2,tag3]

参考文档：

https://hexo.io/zh-cn/docs/front-matter.html
